# 篮球放置挂机：竞品布局原型说明

> 原型版本：v0.2  
> 设计基准：390 × 844竖屏  
> 系统真源：`../GDD.md`  
> 可运行原型：`篮球放置原型.html`

## 1. 本版目标

本版先严格采用《寻道大千》主界面的纵向布局和信息层级，再做篮球题材替换。复制的是结构，不复制竞品美术素材。

上一版把阵容、赛季、管理层、收益等功能分别做成大卡或侧栏，导致主界面拥挤。本版已删除这些额外层级，恢复竞品的单一主景与紧凑槽位结构。

## 2. 竞品结构映射

| 《寻道大千》主界面 | 篮球版本 | HTML模块 |
|---|---|---|
| 顶部头像、名称、战力 | 球队头像、球队名、球队总评 | 顶部球队信息 |
| 仙树与角色活景 | 全场10人训练赛、传球、投篮活景 | 篮球场训练活景 |
| 境界标题 | 球队等级、市值等级、赛季品质 | 球队成长进度 |
| 修为进度条 | 斗志进度条 | 球队成长进度 |
| 12个装备槽 | 12名阵容球员 | 阵容与管理层槽位 |
| 5个长期养成槽 | 5名管理层 | 阵容与管理层槽位 |
| 中央砍树按钮 | 底栏中央招募按钮 | 底部导航与招募 |
| 妖盟、洞府、挑战、冒险 | 联盟、球馆、招募、赛季、更多 | 底部导航与招募 |

## 3. 纵向占比

390×844基准尺寸下：

- 顶部球队栏：约9%。
- 篮球场活景：约46%。
- 等级与斗志进度：约7%。
- 17槽位网格：约22%。
- 招募按钮与底栏：约16%。

这一比例接近竞品实机主界面：中部活景占最大面积，槽位区次之，资源与功能入口均保持紧凑。

## 4. 17槽位网格

槽位区使用6列×3行：

```text
球员1  球员2  球员3  球员4  运营总裁  主教练
球员5  球员6  球员7  球员8  球探总监  队医团队
球员9  球员10 球员11 球员12 媒体团队（横跨2格）
```

这与竞品“12装备槽+5长期养成槽”的结构直接对应，也与当前GDD的12名球员和5名管理层一致。

## 5. 球员表现

- 球场：自动读取当前12人阵容并按OVR倒序，展示最高的10人组成全场训练赛。
- 每次招募上阵后，阵容槽位与训练活景使用同一份名单数据并同时刷新。
- 场上头像角标显示球员OVR，便于确认当前展示顺序。
- 槽位：头像下方只显示纯数字总评，不显示号码。
- 图片加载失败时隐藏图片，槽位结构仍然保留。
- 球场训练只做位移、传球和投篮表现，不影响比赛数值，不代表战术系统。

## 6. 已保留的交互

| 操作 | 原型反馈 |
|---|---|
| 点击中央招募 | 消耗1预算、获得10斗志、弹出球员比较 |
| 点击上阵 | 替换最低OVR槽位并更新球队OVR |
| 阵容发生替换 | 自动重算OVR最高的10名球员并同步到训练活景 |
| 点击球员槽 | 查看球员与阵容规则 |
| 点击管理层槽 | 查看管理层等级、效果与升级 |
| 点击赛季 | 播放约2秒四节比分并结算战绩和预算 |
| 点击球馆 | 查看并领取离线预算 |
| 点击“标注”或按L | 显示/隐藏开发模块标识 |

## 7. 修改入口

HTML中保留以下源码注释：

```html
<!-- 模块：顶部球队信息 -->
<!-- 模块：篮球场训练活景 -->
<!-- 模块：球队成长进度 -->
<!-- 模块：阵容与管理层槽位 -->
<!-- 模块：底部导航与招募 -->
<!-- 模块：功能弹窗 -->
```

常用修改位置：

- 全局颜色：HTML顶部 `:root`。
- 各区域高度：`.app` 的 `grid-template-rows` 与对应模块样式。
- 12名阵容：`ROSTER_MANAGEMENT_GRID` 内的球员槽。
- 5名管理层：JavaScript中的 `management`。
- 场上移动路线：`trainingRoutes`。
- 招募样例：`recruitPool`。
- 比分速度：`runMatchBroadcast()` 内每节等待时间。

## 8. 本版刻意移除

- 主界面阵容大卡。
- 主界面赛季大卡。
- 球场右侧5管理层竖排。
- 离线收益与广告预算的整行双按钮。
- 5项常规底栏。

这些功能没有被删除，只是退回槽位、底栏或二级弹层，避免再次挤压主景。
