## 2. 设计支柱

> MVP阶段核心准则，后续讨论中逐步收敛。

1. **球队老板体验** — 玩家=老板，决策建队方向，而非操控比赛本身
2. **碎片化养成为主** — 招募=核心操作，简单快速，契合微信小游戏场景
3. **赛季循环驱动留存** — 常规赛→季后赛→98–0夺冠→下一品质难度赛季
4. **NBA风味** — 球员品质、属性、评级要有篮球内味

### 4.5 球员属性体系

#### 五围属性

| 属性 | 含义 | 主要参考数据 |
|---|---|---|
| **得分** | 投篮、突破、罚球等进攻能力 | 真实赛季场均得分 |
| **篮板** | 前后场篮板争抢能力 | 真实赛季场均篮板 |
| **助攻** | 传球、组织、大局观 | 真实赛季场均助攻 |
| **抢断** | 防守预判、手速、压迫 | 真实赛季场均抢断 |
| **盖帽** | 护框、垂直弹跳、时机判断 | 真实赛季场均盖帽 |

> NBA历史单场纪录仅作为篮球风味参考，不再作为五项战力的硬上限。

#### OVR先行的属性生成

球员卡不再分别随机五项属性后计算OVR，改为：

1. 每次招募都创建一个独立球员卡实例。
2. 根据球员品质对应的OVR区间，均匀随机出该实例的最终球员OVR。
3. 即使球员原型和品质完全相同，每次招募也会重新随机OVR。
4. 根据真实赛季五项数据、球员位置和球员特点，计算五项属性的分配比例。
5. 将已确定的OVR按比例反推为得分、篮板、助攻、抢断和盖帽。
6. 校验五项属性的最终贡献必须与已随机出的OVR一致。

> 品质负责数值膨胀和随机范围；OVR负责替换爽感；五项属性负责表现球员特色。

#### 重复球员规则

- `data/player_config_fame_v3.json`中的球员记录是当前招募模板，不代表固定数值的唯一卡牌。
- 玩家可以重复获得同一球员、同一品质的球员卡。
- 每张实际获得的卡拥有独立实例ID、独立OVR和依据该OVR生成的五项属性。
- 即使两张卡的球员原型和品质相同，也可能拥有不同OVR，玩家仍需比较后决定如何处理。

#### 招募结果比较规则

- 新球员卡出现后，只与当前12人阵容中OVR最低的球员比较。
- 阵容未满时优先填入空槽位。
- 招募结果界面不同时比较多个球员，避免分散核心正反馈。
- 新球员更强时，重点展示替换后的球队总评绝对增量。

```text
球员基础增量 = 新球员OVR - 阵容最低球员OVR
球队总评增量 = 球员基础增量 ×（1 + 主教练比赛OVR加成）
```

- 点击“上阵”后，新球员自动替换当前阵容中OVR最低的球员。
- 核心反馈重点展示球队总评的绝对增量。

#### 招募结果处理

- MVP阶段不设置球员背包，不允许将未上阵球员暂存。
- 每次招募结果只提供两个处理命令：
  - **上阵**：新球员替换阵容最低OVR球员；被替换的旧球员自动离队。
  - **解雇**：新球员不进入阵容，直接放弃该球员卡。
- 新球员OVR低于阵容最低值时，默认突出“解雇”按钮。
- 球员图鉴只记录“曾经获得”，不承担卡牌库存功能。

#### 位置分配权重

球员位置决定OVR反向分配至五项属性时的初始倾向，再结合真实赛季数据微调。具体权重见 `data/balance/position_attribute_weights.json`。

#### 球员OVR生成公式

```text
球员OVR = 在所属品质的[OVR下限, OVR上限]内均匀随机取整
```

- OVR不再由五项属性二次计算。
- 五项属性由OVR反向分配，不能改变已经确定的球员OVR。
- OVR区间与重叠参数统一读取 `data/balance/player_ovr_ranges.json`。

#### 球队总评公式

```text
球员基础总评 = 12名上阵球员OVR之和
普通球队总评 = floor(球员基础总评 ×（1 + 主教练比赛OVR加成）)

若12名上阵球员全部为概念神：
  每名概念神OVR = 2,147,483,647
  球队总评 = 2,147,483,647
  进入游戏毕业状态，所有比赛直接获胜
```

- 只有主教练参与球队总评计算，其余四名管理层不提供OVR加成。
- 数值计算采用32位有符号整数饱和上限，任何结果不得溢出`2,147,483,647`。
- 全概念神阵容直接进入MAX分支，不执行12人OVR求和或主教练乘算。
- 全概念神阵容的必胜判定优先于对手OVR和随机胜负；拆出任意一名概念神后，MAX与必胜效果立即解除。
- 球员数值或球队总评等于整数上限时，前端统一显示`MAX`。
- 球员替换界面重点展示球队总评的绝对增量。

#### 设计意图

- **品质区间 = 数值天花板**：品质直接决定球员OVR的随机范围
- **真实数据 = 球员特色**：相同OVR的球员也会拥有不同的五项属性结构
- **位置权重 = 属性倾向**：PG更多分配给助攻，C更多分配给篮板和盖帽
- **槽位不分位置 = 完全自由**：位置不提供阵容均衡奖励、错位加成或同位置惩罚
- **比赛不读取位置**：胜负只读取球队OVR；比分表现读取已生成的得分属性，但不会因位置标签再次加成或减益

### 4.6 球员命名规则

> 以真实NBA球员为原型，使用中文姓氏**同音异字**，避开贬义/歧义字。

**选字原则**：
- ✅ 同音或极近音，球迷一眼认出原型
- ✅ 中性/正面字义
- ❌ 动物/昆虫偏旁（蝌、蚪、鼠、猪等）
- ❌ 性别歧义字（母、女等，除非音译必需）

## 5. 核心循环

### 5.1 完整经济循环图

```
预算（在线/比赛/广告/离线）
  ↓ 消耗固定预算
招募球员 ─────────────→ 固定获得斗志
                              ↓
                     斗志满足后点击升级
                              ↓
                         提升球队等级
  ↓
获得球员卡
  ├─ 更强：上阵，旧球员离队
  └─ 更弱：放弃新球员卡

12名球员 + 5名管理层
  ↓
球队总评 = Σ球员OVR ×（1 + 主教练比赛OVR加成）
  ↓
常规赛 → 季后赛
  ↓
单场胜利：预算
  ↓
达到当前球队等级上限
  ↓
进入季后赛并夺冠
  ↓
球队市值提升
  ├→ 切换至下一级球员品质招募概率表
  └→ 赛季全胜成就 / 下一品质赛季
```

### 5.2 三层时间尺度

#### Moment-to-Moment — 招募抽卡

| 要素 | 设计 |
|---|---|
| **操作** | 点击"招募"按钮，消耗固定预算 |
| **反馈** | 卡片弹出动画 + 品质标识（新秀→GOAT） + 总评数值对比弹窗 |
| **固定成长** | 每次招募固定获得斗志，不受百分比加成 |
| **随机奖励** | 获得球员卡；未上阵球员直接放弃 |
| **玩家感受** | "这次能出什么？""还行，轮换球员""卧槽，全明星！" — 盲盒开箱的刺激感 |

> **对标寻道大千**：砍树→弹出装备，保持碎片化快速操作。

**Fun Hypothesis（核心假设）**：
> 如果玩家每次点击招募都能获得即时视觉反馈 + 品质随机性 + 总评对比的锚定效应，那么招募行为本身就能提供持续的正向情绪波动，驱动玩家不断消耗预算。

#### Session Loop — 赛季推进

| 要素 | 设计 |
|---|---|
| **目标** | 打完一个赛季（常规赛 + 季后赛），冲击冠军 |
| **操作** | 招募抽卡 → 替换阵容 → 反复优化总评 → 模拟比赛 → 查看赛果 |
| **张力** | 总评不够时会卡在当前比赛——激发"再来一个强力球员就过了"的欲望 |
| **验证** | 固定难度表验证玩家本轮成长后的突破距离：上次卡在战力99，成长后可以一路平推到战力999，直到撞到新的固定门槛 |
| **结算** | 每场胜利即时获得预算；98场全胜后获得对应品质全胜成就并开启下一难度；若已达到球队等级上限，同时提升球队市值 |

具体赛季局数、模拟时长和现实时间节奏见 `data/balance/season_pacing.json`。

#### Long-Term Loop — 市值提升 & 建队

| 要素 | 设计 |
|---|---|
| **进度** | 招募积累斗志→斗志满足后点击升级→球队等级提升；达到阶段上限并夺冠→球队市值提升 |
| **预期节奏** | 前期升级较快，后期逐步放缓 |
| **长线目标** | 阵容总评成长 → 完成13个98–0品质赛季 → 进入概念神无限赛程 → 凑齐12名概念神毕业 |
| **留存钩子** | 免费招募 + 广告奖励 + 赛季成就累积（总冠军数/GOAT收集） |

### 5.3 资源经济体系

#### 核心资源

| 资源 | 定位 | 来源 | 用途 | 对标 |
|---|---|---|---|---|
| **预算** | 体力/主货币 | 在线挂机 + 比赛胜利 + 离线挂机 | 招募球员 + 升级管理层 | 仙桃 |
| **球员卡** | 核心养成物件 | 招募 | 填入阵容；未上阵则放弃 | 装备 |
| **管理层卡** | 被动加成物件 | 初始解锁，消耗预算升级 | 填入管理槽位 | 灵兽/精怪/神通/血脉/座驾 |
| **斗志** | 经验值 | 每次招募固定获得 | 提升球队等级 | 修为 |
| **球队等级** | 高频成长等级 | 斗志达到当前等级需求后，由玩家点击升级按钮提升 | 达到阶段上限后开放市值提升条件 | 修为等级 |
| **球队市值** | 阶段成长等级 | 达到球队等级上限后夺得总冠军 | 决定完整品质招募概率表和球队等级上限 | 境界/仙树等级的篮球化合并 |

#### 预算经济闭环

- 预算是核心循环货币，只用于招募球员与升级管理层。
- 预算来源分为三类：在线挂机、比赛奖励、离线挂机。
- 在线挂机与比赛奖励服务短会话，离线挂机服务回流；标准赛季夺冠不再额外发放预算。
- 招募消耗固定预算，并固定获得斗志；广告不直接赠送斗志，但“免费招募”完整执行一次正常招募，因此照常获得10斗志。
- 冗余球员离队不返还预算，避免“抽卡—分解—再抽卡”的闭环稀释招募成本。
- 广告只提供免费招募、强制胜利、直接预算和离线收益翻倍。
- 运营总裁只加成在线预算、比赛预算和广告预算奖励，不加成普通赛季夺冠结算；媒体团队只加成离线预算。
- 管理层升级消耗预算，五个管理层共用一套1–520级升级成本曲线；具体加成效果读取 `data/balance/management_effects.json`。

> 预算产出、消耗、广告次数与管理层升级成本只维护在 `data/balance/economy.json`。

#### 斗志升级与市值提升

- 每次招募固定获得斗志，具体数值读取 `data/balance/economy.json`。
- 斗志达到当前球队等级需求后，升级按钮进入可点击状态；玩家点击后球队等级提升1级并扣除本级所需斗志，多余斗志保留。
- 每次点击只提升1级；若剩余斗志仍满足下一级需求，升级按钮继续保持可点击，玩家可再次点击升级。
- 斗志不足时升级按钮置灰；可升级时按钮亮起并播放轻微提示动效。
- 普通球队等级提升不要求打比赛。
- 球队市值共130级，每级球队市值包含4个球队等级小段，共520个球队等级。
- 每级球队市值内的4个球队等级小段分别对应：季后赛首轮、次轮、分区决赛、总决赛。
- 达到当前球队市值阶段的球队等级上限后，即使斗志满足下一等级需求也不能继续升级，进入“待夺冠”状态；升级按钮改为“去夺冠”。
- 待夺冠状态下进入季后赛并夺得总冠军，球队市值提升1级，同时解锁下一组4个球队等级。
- 若季后赛未夺冠，球队等级保持在阶段上限，玩家继续招募强化阵容并再次挑战。
- 市值提升后球队等级不重置，升级按钮恢复可用；玩家仍需点击升级按钮，才会消耗已积累斗志并进入下一组球队等级。
- 球队市值共130级，每一级球队市值对应一张完整的品质招募基础权重表。
- 每10级球队市值对应一个品质段；每一级至少5档品质可招募，品质池随市值上滑，口径对齐《寻道大千》。
- 1–10级可招募新秀、饮水机、轮换、第六人、首发；11–20级可招募饮水机、轮换、第六人、首发、核心；依次上滑。
- 进入下一段后，上一窗口最低品质退出普通招募。例如11级开始不再通过普通招募获得新秀品质。
- 81级后进入最终五档池：MVP、FMVP、名人堂、传奇、GOAT；后续市值提升继续提高高档权重。
- 招募时先读取当前球队市值的概率表并抽取品质，再从该品质球员名单中随机球员。
- 某品质在当前市值下无掉落概率时，即视为尚未解锁；不额外设置独立的品质解锁系统。
- 球队市值提升后立即切换至下一级概率表；每10级进入一个新的品质段。
- 同一球队市值下的概率固定，不受球队总评、阵容球员或管理层百分比二次修改。
- 球探总监只对当前可招募池内的最高品质追加权重，不扣低品质；最终权重总和允许超过100，并按最终权重池抽取。
- 等级数量、逐级斗志需求及阶段划分参考《寻道大千》原作成长表，不额外加入百分比斗志加成。

> 招募概率的具体数值只维护在 `data/balance/recruitment_probability.json`。球队等级与斗志需求只维护在 `data/balance/team_progression.json`。

#### 冗余球员处理

```
招募 → 获得球员卡
  ├── 比现有槽位好 → 替换上场（正向情绪）
  │     └── 被替换的旧球员自动离队
  └── 不替换 → 直接放弃新球员卡
```

> MVP阶段没有球员背包，也不设置冗余球员回收货币。招募结果必须立即选择上阵或放弃，保持“招募→比较→上阵或解雇”的快速循环。

### 5.4 赛季与比赛系统

#### 设计原则

- **纯 PVE**：对手为 AI 球队，OVR 按赛程线性递增
- **简单规则**：比赛结果由双方 OVR 比值直接决定，无额外操作
- **分节播报**：MVP 阶段展示四节比分滚动，后续加入文字描述
- **突破距离验证**：82 场常规赛不是可跳过的批量结算，而是在固定难度表上验证玩家成长后能平推多远
- **全胜幻想**：正式战绩只记录已通关的胜场；失败代表当前比赛尚未通过，玩家强化后从原地重打，最终建立常规赛82–0、季后赛16–0的无敌赛季

#### 单场比赛胜负规则

```
若当前12名上阵球员全部为概念神 → 直接获胜，跳过以下OVR与随机判定

玩家OVR ≥ 对手OVR × 1.1  →  必胜
玩家OVR ≤ 对手OVR × 0.9  →  必败
对手OVR × 0.9 < 玩家OVR < 对手OVR × 1.1  →  50% 胜率
```

> 全概念神阵容拥有最高胜负优先级。其他阵容每次挑战独立判定，没有连败保底或连胜利制；挑战失败不会写入正式赛季战绩。

**位置与属性口径：**

- 单场胜负只由双方球队OVR决定，PG/SG/SF/PF/C位置字段不参与计算。
- 不设置首发位置要求、位置均衡奖励、错位克制、同位置惩罚或阵容合法性检查。
- 12名球员可以全部来自同一位置，不影响球队OVR和胜负判定。
- 五项属性不额外修改胜负；其中得分属性只用于生成展示比分，其余属性暂用于体现球员特色。

#### 比分生成规则

执行顺序：**先定胜负 → 再算比分**

**Step 1 — 双方原始得分**

```
球队原始得分 = floor((12人得分属性总和 / 12) × 5 × 得分转比分系数 × random(0.90, 1.10))
得分转比分系数 = 0.1
比分无上限
```

> 单场按场上5人等效计算，再乘0.1显示倍率。新秀局接近现实篮球比分；场上全是GOAT或概念神时，仍可以自然出现几万、几亿分。

**Step 2 — 按胜负结果修正分差**

根据胜负结果和 OVR 差距反推：

```
OVR差距率 = abs(玩家OVR - 对手OVR) / max(玩家OVR, 对手OVR)
必要分差 = ceil(max(1, 胜方原始得分 × (1% + OVR差距率 × 25% + random(0%, 3%))))

若双方处于0.9–1.1不确定区间：
  必要分差上限额外限制为胜方原始得分 × 12%

若 胜方原始得分 > 负方原始得分：
  胜方最终得分 = 胜方原始得分
  负方最终得分 = 负方原始得分

若 胜方原始得分 <= 负方原始得分：
  胜方最终得分 = 负方原始得分 + 必要分差
  负方最终得分 = 负方原始得分
```

> 胜负仍由 OVR 规则先决定；比分只负责表现。若原始得分和已判定胜负冲突，就抬高胜方分数，确保比分不反向推翻胜负。

**Step 3 — 四节分配**

```
将双方得分按四个随机比例拆分为四节：
  每节占比 = 25% ± 7%波动，四节之和归一化
  归一化后四节整数分之和必须等于全场总分
```

> 产出每场四组节比分，两秒滚动播报。比分只做结果表现，不反向修改胜负。

#### 逐场挑战交互

- 点击“继续赛季”后从当前未通关比赛开始，可自动逐场进行；每次挑战都完整播放约2秒四节比分，不可跳过或扫荡。
- 挑战胜利：结算本场胜利奖励、正式战绩增加1胜、保存进度并自动进入下一场。
- 挑战失败：不计入正式战绩、不给比赛奖励、不推进赛程，自动暂停在当前比赛。
- 失败界面展示本队OVR、对手OVR、双方比值、当前所处胜负区间，以及达到必胜线仍需提升的OVR。
- 玩家可在失败后进行招募、替换阵容和升级管理层，再点击“重新挑战”原场次。
- 不强制检测阵容是否成长后才能重试；但处于必败区间时，双方OVR不变就仍然必败，玩家必须强化才能通过。
- 玩家也可在任意比赛中点击“本场结束后暂停”；正在播放的本场不会被中断，结算后停留在下一场赛前。
- 退出游戏会保存当前已通关场次；再次进入时从当前未通关比赛继续。
- 一次完整冠军赛季固定需要98个正式胜场：常规赛82胜 + 季后赛四轮各4胜；正式败场始终为0。

**单场奖励规则：**

- 每次挑战胜利后立即结算本场预算，失败不给奖励。
- 单场基础预算与本场对手OVR成正比，以新秀赛季第1场对手OVR 696作为1预算基准：

```text
单场基础预算 = max(1, ceil(本场对手OVR ÷ 696))
单场最终预算 = ceil(单场基础预算 × (1 + 运营总裁加成))
```

- 常规赛与季后赛使用同一公式，不再额外叠加季后赛轮次奖、系列赛奖或总冠军预算奖；比赛本身更难，奖励自然更高。
- 13个品质赛季共1274场的对手OVR与基础预算逐场结果见 `data/balance/match_rewards.json`。
- 概念神无限赛程继续沿用同一公式，待无限赛程对手OVR生成后即可直接计算。
- 全概念神阵容自动获胜时仍按本场对手OVR正常结算预算，比赛编号与赛程正常推进。

#### 赛季循环与品质难度

- 赛季难度是独立于球队市值的长期推进线，按已完成的冠军赛季数递增，不会因球队市值未提升而重复旧难度。
- 初始为赛季1“新秀难度”；完成98–0并夺冠后，正式战绩重置为0–0，立即开启赛季2“饮水机难度”。
- 后续依次为：轮换、第六人、首发、核心、全明星、最佳阵容、MVP、FMVP、名人堂、传奇、GOAT，共13个已配置赛季。
- 每个品质赛季的总决赛对手OVR固定为该品质单卡OVR上限×12；常规赛与季后赛各节点按固定比例生成。
- 相邻赛季之间不回落：新赛季第1场对手OVR必须高于上一赛季总决赛对手OVR，因此进入下一品质赛季后，从常规赛开局就能感受到难度跃迁。
- 赛季通关只推进赛季难度；球队市值仍按“球队等级达到当前阶段上限后夺冠”独立判断是否提升。一次夺冠可以同时推进下一赛季并完成市值提升，也可以只推进赛季。
- 新赛季保留玩家现有阵容、预算、斗志、球队等级、球队市值和管理层，仅重置本赛季正式战绩及比赛进度。
- GOAT难度赛季98–0夺冠后，不再开启新的标准赛季，而是永久进入“概念神无限赛程”。

#### 概念神无限赛程

- 解锁条件：完成GOAT难度赛季98–0并夺冠。
- 进入后取消赛季概念，不再区分常规赛、全明星前后、季后赛、系列赛和总决赛。
- 比赛编号从概念神第1场开始持续累加，没有最大场次，也不会周期性重置。
- 每场胜利后难度继续提升并进入下一场；失败不计战绩、不给奖励、不推进编号，停在当前场等待强化后重打。
- 概念神无限赛程不再产生“赛季夺冠结算”，只结算单场胜利奖励；具体里程碑奖励后续随概念神经济一起确定。
- 无限赛程的对手OVR在达到`2,147,483,647`前必须随场次严格递增；达到整数上限后保持MAX，比赛编号仍可无限推进。
- 无限赛程起始OVR和从起点增长至MAX的公式仍待配置。
- 当前阵容集齐12名概念神后进入游戏毕业状态：此后的所有比赛直接获胜，跳过OVR区间和随机判定，但仍播放比赛结果、推进比赛编号并正常结算单场预算。
- 毕业状态下不出现强制胜利广告。毕业成就一经触发永久保留；若之后拆出任意一名概念神，MAX与自动获胜效果立即失效，但毕业成就不撤销。

#### 常规赛（82场）

- 82 场逐一进行，不可跳过或扫荡
- 每一场比赛都承担“固定难度表上的突破距离验证”作用：玩家成长后，应能明显平推之前卡住的一段赛程，直到撞到新的固定战力门槛。
- 对手 OVR 按当前赛季难度等级读取 `data/balance/season_pacing.json` 中的固定难度表，并在82场内从起始 OVR 线性递增至结束 OVR。
- 82场按全明星周末分为两个展示阶段：全明星前为第1–41场，全明星后为第42–82场。
- 全明星周末只作为赛程节点，不重置或降低难度；第1场至第82场保持同一条线性增长曲线，第42场难度高于第41场。
- 每个品质赛季都有独立的常规赛起止 OVR 与季后赛四轮 OVR；对手 OVR 使用球队总评单位，而非单卡 OVR 单位。
- 赛季难度是固定关卡表，不根据玩家当前阵容、招募概率、斗志需求或预期成长动态重算。
- 若玩家上次卡在战力99附近，经过招募和管理层成长后能一路平推到战力999附近，说明本轮成长被固定表清晰验证；新的卡点会成为下一轮强化目标。
- 每场约 2 秒显示四节比分滚动播报
- 只有获胜才推进场次；失败挑战不计入82场正式赛程
- 通关全部82场后，常规赛正式战绩固定为82胜0负，并以头号种子身份直接进入季后赛
- MVP阶段不再计算常规赛排名、净胜分排名或“前16名”资格

#### 季后赛

- 常规赛82–0通关后直接进入季后赛。
- 季后赛四轮对手强度分别对应：季后赛首轮、次轮、分区决赛、总决赛。
- 季后赛难度在常规赛第82场之后继续严格递增，固定为当前赛季总决赛门槛的85% → 90% → 95% → 100%，不会出现进入季后赛后难度回落。
- 总决赛 OVR 是当前品质赛季的固定冠军门槛；玩家通过招募、阵容替换和管理层强化去追赶这张表。
- 每轮需要连续通关4个正式胜场；失败挑战不计入系列赛比分，强化后重打当前场，因此最终系列赛正式比分固定为4–0
- 四轮全部通关后，季后赛正式战绩为16胜0负
- 赛前可选择战术倾向：
  - **全力进攻**：OVR × 1.05 但防守减益（暂不实现，预留接口）
  - **均衡**：默认
  - **铁血防守**：OVR × 1.05 但进攻减益（暂不实现，预留接口）
- MVP 阶段战术系统暂不实现，所有比赛使用均衡模式

### 5.5 管理层系统

> 对标寻道大千右侧 5 槽位（灵兽/精怪/神通/血脉/座驾），每个管理层独立获取并产出专项加成。

#### 5.5.1 槽位与职能

| # | 管理层 | 对标竞品 | 职能 | 加成效果 |
|---|--------|---------|------|---------|
| 1 | **运营总裁** | 血脉 | 预算运营 | 只提升在线预算收益、比赛预算奖励、广告预算奖励，不影响离线收益 |
| 2 | **主教练** | 神通 | 比赛战术 | 只提供比赛 OVR 百分比加成，计入球队总评公式 |
| 3 | **球探总监** | 灵兽 | 招募品质 | 只提升当前球队市值下**最高已解锁品质**的招募概率权重 |
| 4 | **队医团队** | 精怪 | 球员养成 | 只在招募获得新球员卡时，对 OVR 随机结果提供正向偏移；不改变品质区间上下限，但在区间内偏向上限取值 |
| 5 | **媒体团队** | 座驾 | 知名度收益 | 提升离线预算收益倍率 |

#### 5.5.2 球探概率加成口径

- 球探总监对最高已解锁品质提供固定加成，例如当前最高已解锁品质概率权重为 3，球探总监提供 +1，则该品质权重变为 4。
- 该加成不从低品质权重中扣除，允许整张概率表的权重总和超过 100。
- 招募抽取时按最终权重池进行随机，不要求各品质显示值重新归一为 100%。

**前端展示规则：**

- 招募主界面统一使用“权重”一词，不把权重写成“概率”或添加百分号。
- 默认只展示当前可招募的5档品质，按品质从低到高排列；未进入当前招募池的品质不占主界面空间。
- 每档品质主行展示“品质名称 + 最终权重”；当前最高品质额外显示“最高品质”文字标签和球探图标，不只依靠颜色区分。
- 点击权重区域后展开详细拆分：`基础权重 + 球探加成 = 最终权重`。非最高品质的球探加成为0，可弱化显示。
- 列表底部固定说明：“权重用于相对抽取，不等于百分比；总权重可以超过100。”
- MVP阶段不展示归一化后的实际概率，避免“权重”和“概率”两套数字同时出现。
- 球探升级界面展示下一等级预览，例如：`最高品质权重 +2.40 → +2.41`；升级成功时只对最高品质行播放一次短促数值跳动。
- 所有权重保留两位小数；触控展开区域不小于48×48dp。

#### 5.5.3 管理层获取与升级

- 管理层不通过抽卡获得，初始解锁后永久持有。
- 五个管理层初始均为1级，各自独立升级，最高均为520级。
- 每个管理层的当前等级不得超过当前球队等级：`管理层可升级上限 = min(当前球队等级, 520)`。
- 球队升级只负责开放更高的管理层等级上限，不会免费提升管理层；玩家仍需分别消耗预算升级五个职位。
- 若管理层已达到当前球队等级，则升级按钮锁定并提示“请先提升球队等级”。
- 所有五项管理层的升级均消耗 **预算**（与招募消耗同一货币）。
- 五个职位共用升级成本公式：`ceil(5 × 1.015^(当前管理层等级-1))`，从1升至520共519次升级。
- 单个职位1→520级总成本为756,339预算，五个职位全部满级总成本为3,781,695预算。
- 各级升级成本见 `data/balance/economy.json`；各管理层的逐级效果见 `data/balance/management_effects.json`。

#### 5.5.4 管理层数值曲线

| 管理层 | 每级效果 | 520级效果 | 生效范围 |
|---|---:|---:|---|
| 运营总裁 | 预算收益 +0.2% | +104% | 在线预算、正式比赛胜利预算、广告预算奖励 |
| 主教练 | 比赛OVR +0.05% | +26% | 球队比赛OVR |
| 球探总监 | 最高品质权重 +0.01 | +5.20 | 当前招募池最高品质 |
| 队医团队 | 新卡OVR随机位置 +0.05个百分点 | +26个百分点 | 品质OVR区间内部随机，不突破上下限 |
| 媒体团队 | 离线预算 +0.4% | +208% | 离线预算收益 |

- 五项效果均按管理层等级线性增长，升级前后差值固定，便于玩家理解。
- 队医计算方式：`最终随机位置 = min(100%, 原始均匀随机位置 + 队医偏移)`，再映射回当前品质OVR区间。
- 运营总裁不影响离线预算和普通赛季夺冠结算；媒体团队不影响在线、比赛或广告收益。

#### 5.5.5 与球队总评公式的关系

```
球员基础总评 = Σ12人 OVR
管理层总评加成 = Σ(主教练加成百分比)  ← 只有主教练计入总评
球队总评 = 球员基础总评 × (1 + 管理层总评加成)

运营总裁、球探总监、队医团队、媒体团队的加成不通过总评公式生效，
而是直接作用于各自对应的经济环节。
```

> 设计意图：五个管理层各管一摊，互不重叠。主教练管战力，其他四个管经济/养成/收益——玩家的管理层升级优先级决定了球队发展策略方向。

### 5.6 IAA 广告点位布局

| 广告点位 | 触发时机 | 规则 |
|---|---|---|
| **免费招募** | 招募界面 | 不消耗预算，完整执行一次正常招募，照常获得球员卡与10斗志 |
| **强制胜利** | 0.9–1.1不确定区间随机判负后 | 将本次随机败局改判为胜利，推进正式赛程，并照常获得该场预算 |

- 玩家OVR≥对手OVR×1.1时本来就必胜，同样不显示强制胜利广告。

- 直接预算广告的基础奖励从球队1级的10预算线性增长至球队520级的5200预算；运营总裁满级后最高为10608预算。
- MVP阶段不再提供在线预算加速、球探集中考察、赛季结算翻倍或季后赛续命广告。

## 7. 概念神系统

> 概念神是独立于13档普通品质的特殊品质。它不进入普通招募权重表，而是通过累计招募GOAT触发保底。

### 7.1 设计目标

- **强化招募循环** —— 概念神仍然来自招募，不额外搭建觉醒或任务系统
- **有梗味** —— 48个概念神保留各自名称、原型和梗解释
- **稀缺感** —— 获得概念神是游戏的中长期目标，而非即时满足
- **可规划** —— 玩家始终能看到距离下一张概念神还差多少张GOAT

### 7.2 核心机制：GOAT计数保底

每轮概念神保底开始时锁定：

```text
X = 本轮开始时当前12人阵容中的概念神数量 + 1
本轮所需GOAT数量 = 100 × X
```

- 付费招募与广告免费招募获得的GOAT都使本轮进度+1。
- X在本轮开始时锁定，中途调整阵容不会改变当前门槛。
- 达到`100×X`张GOAT时，该张GOAT正常获得；下一次招募才强制改为概念神。
- 保底概念神使用玩家下一次选择的正常招募方式：付费招募照常消耗预算，免费招募照常免费。
- 概念神保底结果不算作GOAT，也不受普通品质权重或球探加成影响。
- 概念神出现并完成“上阵或解雇”选择后，本轮计数归零，并按处理后的当前阵容重新锁定下一轮X。
- 概念神不需要原型球员卡、任务条件、比赛里程碑或其他货币。
- 前端固定展示：`概念神保底：当前GOAT数量 / 100×X`。

| 当前阵容概念神 | X | 下一张概念神前需要招募的GOAT |
|---:|---:|---:|
| 0 | 1 | 100 |
| 1 | 2 | 200 |
| 2 | 3 | 300 |
| … | … | … |
| 11 | 12 | 1200 |
| 12 | 13 | 1300 |

> 从0张开始凑齐12张概念神阵容累计需要7800张GOAT。完整逐级门槛见 `data/balance/concept_god_recruitment.json`。

### 7.3 概念神招募池

- 当前共48个概念神，结构化数据见 `data/concept_god_pool.json`，原始梗素材继续保留在 `data/nba_concept_gods.md`。
- 保底触发时从概念神池等权随机；概念神之间暂不设置稀有度权重。
- 当前阵容中已有的同名概念神从本次抽取池排除，避免阵容内重复。
- 曾经获得但已解雇的概念神可以再次出现，避免无背包规则造成永久丢失。
- 同一原型球员可以对应多个独立概念神，例如“替补大笑邓”和“大风车邓肯”分别作为两张不同卡。
- 概念神招募结果继续沿用普通招募的即时处理：与阵容最低OVR球员比较后选择上阵或解雇，不进入背包。

### 7.4 概念神数值设计

概念神没有固定OVR区间。其OVR和五项属性由当前阵容动态计算。

```text
C = 本次阵容变更完成后的概念神数量

当 1 ≤ C < 12：
  对每个字段分别计算：
  非概念神字段最高值 = max(当前阵容所有非概念神的该字段实际值)
  所有概念神的该字段值 = floor(非概念神字段最高值 × (1 + 0.1 × C))

字段包括：
  OVR、得分、篮板、助攻、抢断、盖帽

当 C = 12：
  所有概念神的上述六项数值 = 2,147,483,647
  球队总评 = 2,147,483,647
  所有比赛直接获胜，进入游戏毕业状态
```

- 第1名概念神入阵后倍率为1.1，第2名为1.2，依次提升；第11名为2.1。
- 新概念神入阵前的`X=原阵容概念神数量+1`，等价于入阵后的C，因此与GOAT保底公式中的X口径一致。
- 每个字段独立寻找非概念神最高值；最高OVR、最高得分和最高篮板可以来自不同球员。
- 同一阵容中的所有概念神在同一字段上数值相同，概念神之间的差异由名称、原型、梗说明和后续视觉表现承担。
- 计算结果向下取整，并以32位有符号整数上限进行饱和保护。
- 每次任何球员上阵、替换、离队或读取存档后，都重新计算全部概念神数值；新概念神入阵一定触发刷新。
- 若概念神离开全概念神阵容，MAX状态立即解除，并按照当前剩余非概念神重新计算。
- 首次凑齐12名概念神时永久记录“概念神全阵容·游戏毕业”成就；自动获胜效果只在当前阵容仍为12名概念神时生效。
- 详细字段、刷新顺序与MAX判定见 `data/balance/concept_god_values.json`。

| 参数 | 取值 |
|---|---|
| **品质等级** | 特殊品质，独立于Lv3-Lv15体系，不参与普通招募概率 |
| **颜色** | 专属概念神色（待定） |
| **数值来源** | 当前阵容非概念神各字段最高值 × 概念神数量倍率 |
| **在阵容中** | 占一个常规球员槽位，与其他品质球员共存 |
| **阵容内重复** | 同名概念神不能重复上阵；已解雇的概念神可以再次招募 |
| **全概念神阵容** | 球员六项数值和球队总评真实值均为`2,147,483,647`，显示为MAX；所有比赛直接获胜 |
| **终局用途** | 达成游戏毕业；无限赛程仍可继续推进并领取单场奖励 |

### 7.5 设计意图

- **单循环驱动**：玩家不需要离开招募界面处理另一套任务或觉醒系统
- **保底可见**：GOAT累计进度始终明确，概念神稀有但不是纯随机
- **不挤压普通品质**：概念神不进入普通权重池，只在保底就绪后的下一次招募中出现
- **长期驱动**：48个概念神=48个长期目标，对应用户的48种追求路径
- **终局验证**：概念神无限赛程不再重复标准赛季结构，直接用无上限难度持续验证终局阵容能推进多远
